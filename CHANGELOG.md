# Changelog

All notable changes to this project are documented here.

Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/); versions
follow [semver](https://semver.org/), with the CLI-specific meaning recorded in
[docs/07](docs/07-distribution-and-roadmap.md#2-versioning--the-template-drift-problem):
**patch** = fixes, **minor** = new commands or template capabilities, **major** =
a convention change that would make `doctor` disagree with projects generated by
an older version.

## [1.0.0] — unreleased

Published as **`jinn-web`**. The architecture it generates is Codeable's
convention set — the tool is a personal project and its name says so.


First complete release. Everything below has been verified by generating a
project and running the full gate on it, not by inspection.

### Commands

- **`create`** — scaffold a project with per-project roles, locales, brand
  colour and API URL. Verifies its own output with `tsc` before handing it over.
- **`domain` / `remove-domain`** — a full CRUD domain (nine files) wired into
  six registries: routes, query keys, nav, the endpoint index and every locale
  catalog. Removal is the exact inverse, asserted by hashing the whole tree.
- **`role` / `remove-role`** — extend `ROLES` and every map keyed off it, then
  run `tsc` and report the remaining errors as a decision checklist. Removal
  refuses while the role still owns domains.
- **`component`** — placed at the right layering tier (domain → role `_shared`
  → cross-role `components/shared`).
- **`dialog`** — confirmation or custom, inside a domain.
- **`add-locale`** — seeds a catalog from the source language with every string
  marked `TODO(<code>)`, and registers it.
- **`rename`** — package and display name.
- **`doctor`** — five static checks over the wiring generators create, with
  `--fix` for the mechanical ones and `--json` for CI.
- **`guardrails`** — install the architecture lint rules into any project,
  including one that predates the CLI. `--level warn` to adopt gradually.

### Template

Next.js 16 · React 19 · TypeScript strict · Tailwind v4 tokens (dark-ready) ·
shadcn/ui · TanStack Query + Table · Zustand · Zod · nuqs URL state · typed i18n.

- **BFF cookie auth** — refresh token in an httpOnly cookie via a thin
  `/api/session` handler, access token in memory. Enables server-side gating in
  `proxy.ts` and keeps a long-lived credential out of reach of XSS.
- **Role-first features** with exhaustive `Record<Role, …>` maps and declarative
  `<RoleScreens>` dispatch.
- ~35 shared components, 11 form fields, a DataTable system on URL state, and a
  helper layer for dates, formatting, mutations and reporting.
- ESLint guardrails, Vitest units, and a Playwright sweep for the defects static
  analysis can't see — layout shear, unreachable truncation, unnamed controls.

### Notable fixes found by verifying output rather than reading code

- The message catalog was `as const`, making every value a literal type, so no
  second locale could satisfy `Messages`. Only visible once a project was
  generated with two languages.
- The brand ramp anchored on the nearest step by lightness, so `#2563EB` landed
  on 700 and `#10B981` on 500 while `--primary` maps to `brand-600` — the app's
  primary colour was never actually the brand colour.
- `TruncatedText` measured itself correctly, set its clipped flag, then
  re-parented the span into the tooltip trigger; that detached the node its
  `ResizeObserver` watched, which fired at 0×0 and reset the flag. Truncated
  values silently never got a tooltip.
- `addNamedImport` rebuilt import lists from names alone, dropping the `type`
  modifier and turning type-only imports into value imports.
- `guardrails` asked ESLint for a formatter removed from core, then read the
  empty output as "no violations" — a false green in a tool whose job is finding
  problems.
- CI ran `npm ci`, which requires the lockfile to describe the install target
  exactly; a lockfile written on macOS/arm64 doesn't carry every Linux optional
  binary, so every run was red from the day the pipeline landed.

### Known limitations

- Publishing to npm is not yet done; install from the repository.
- The template ships no reset-password screen (the backend contract for it
  varies too much to guess).
- `doctor`'s catalog parity check is a belt to TypeScript's braces — it parses
  catalogs textually, so a deliberately exotic catalog shape could confuse it.
